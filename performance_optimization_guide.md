# THE3 Studio 시네마틱 영상 성능 최적화 가이드 🚀

현재 THE3 Studio의 메인 히어로 영상과 갤러리 배경 영상이 끊기거나 로딩이 느린 문제를 해결하기 위해 **코드 최적화**와 **파일 최적화** 두 가지 단계를 진행합니다.

## 1. 지금까지 적용된 코드 최적화 (Done ✅)

제가 코드 레벨에서 다음의 기능들을 이미 적용해 두었습니다:

- **Intersection Observer 도입**: 화면에 보이지 않는 영상은 자동으로 일시정지(`pause`)하여 CPU/GPU 자원을 보존합니다.
- **GPU 가속 활성화**: CSS `will-change: transform` 속성을 통해 브라우저가 영상을 렌더링할 때 하드웨어 가속을 사용하도록 설정했습니다.
- **포스터 이미지 예약**: 영상이 로드되기 전 검은 화면 대신 보여줄 이미지(`poster`) 경로를 설정했습니다.
- **캐시 버스팅**: 브라우저가 이전의 무거운 영상을 기억하지 않고 새로운 최적화 버전을 불러오도록 버전 번호를 `v=1.3`으로 업데이트했습니다.

---

## 2. 사장님이 해주셔야 할 작업 (Action Required 🚨)

영상 파일 자체가 너무 크면(예: 30MB 이상) 아무리 코드를 고쳐도 데이터 전송 속도 때문에 끊길 수 있습니다. 아래 단계에 따라 영상을 더 가볍게 만들어주세요.

### A. 영상 압축 및 포맷 변형 (강력 권장)
가장 좋은 방법은 **WebM** 포맷을 사용하는 것입니다. MP4보다 훨씬 압축률이 좋습니다.

1. **온라인 도구 활용**: [CloudConvert](https://cloudconvert.com/mp4-to-webm) 또는 [Handbrake](https://handbrake.fr/) 같은 무료 도구를 사용하세요.
2. **설정값**:
   - **해상도**: 1920x1080 (FHD) 유지
   - **프레임**: 24fps 또는 30fps (시네마틱한 느낌을 위해 24fps 권장)
   - **비트레이트**: `2Mbps ~ 4Mbps` 정도면 충분히 고화질로 보입니다. (현재보다 파일 크기를 1/10로 줄이는 것이 목표입니다.)
3. **파일명**: 압축된 파일을 기존 파일명(`hero-desktop.mp4`, `hero-mobile.mp4` 등)으로 덮어쓰거나, 새로운 파일로 업로드 후 제가 수정해 드릴 수 있습니다.

### B. 포스터 이미지 생성
영상이 뜨기 전 0.5~1초 동안 자연스럽게 보이도록 영상의 첫 프레임을 캡처해서 아래 경로에 저장해주세요.

- **PC 히어로**: `images/hero-poster-desktop.jpg`
- **모바일 히어로**: `images/hero-poster-mobile.jpg`

## 3. 요약 및 기대 효과
이 설정이 모두 완료되면:
1. 페이지 접속 시 **첫 화면(LCP)**이 즉시 나타납니다.
2. 스크롤을 내려 영상이 안 보일 때 브라우저가 쉬게 되어 **전체 사이트 반응 속도**가 올라갑니다.
3. 데이터 사용량이 줄어들어 **모바일 환경**에서의 사용자 경험이 극적으로 개선됩니다.

작업 중 궁금한 점이 있으시면 언제든 말씀해 주세요! 😊
